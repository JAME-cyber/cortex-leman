🔐 Architecture de Sécurité

Médiateur Déterministe

Le composant central de Cortex Leman (Médiateur) est 100% déterministe (JsonLogic). Aucun LLM probabiliste n'est utilisé pour les décisions critiques. Les règles sont explicites, auditable et immuables.

Journal WORM (Write Once, Read Many)

Chaque décision est horodatée et hash-chainée (SHA-256). Le journal est inviolable — tout audit peut vérifier l'intégrité complète de l'historique.

Mode Haute Protection

Pour les professions réglementées (avocat, banque, santé), le Mode Haute Protection active :

  • Zero appel réseau externe (Ollama local uniquement)
  • Chiffrement AES-256 des données au repos
  • Data residency garantie en Suisse ou en Europe
  • Arbitrage humain obligatoire pour tout conflit

Circuit Breaker & Saga

Chaque agent dispose d'un circuit breaker indépendant. En cas de défaillance, le mode dégradé s'active automatiquement avec compensation (Saga pattern).

📊 Traitement des Données

Catégories de données traitées

  • Données d'identification — nom, email, organisation (base contractuelle)
  • Requêtes réglementaires — questions professionnelles (base légale)
  • Métadonnées de décision — scores de confiance, statut (obligation légale)
  • Logs d'audit — métadonnées uniquement, pas de contenu métier

Sous-traitants

  • Mode Standard : OpenRouter (US) — DPA en place, transfert encadré
  • Mode Haute Protection : Ollama (local) — zero transfert, zero sous-traitant

Durées de conservation

  • Journal WORM : 5 ans (obligation légale)
  • Données d'intention : durée contractuelle + 2 ans
  • Logs d'audit : 5 ans
  • Données personnelles : suppression sur demande (RGPD Art. 17)

🤖 Transparence IA

Conformément à l'AI Act (Art. 52) et au RGPD, Cortex Leman :

  • Informait explicitement chaque utilisateur de l'utilisation d'IA dans le processus
  • Ne prend jamais de décision finale sans intervention humaine (arbitrage)
  • Maintient un registre public de ses règles de médiation (endpoint /trust-box/rules)
  • Fournit un serment numérique accessible (/trust-box/serment)
  • Permet à tout utilisateur de demander une explication sur toute décision IA (RGPD Art. 22)

🔌 Intégration MCP

Notre conformité est accessible via Model Context Protocol. Tout agent IA peut vérifier une action en temps réel en appelant notre MCP Server.

Outils disponibles

  • check_compliance — Vérifier la conformité d'une action (verticale + type d'action)
  • estimate_risk — Estimer risque + coût de non-conformité (amendes, frais)
  • list_verticals — Lister les 7 verticales avec règles
  • get_rules — Obtenir les règles détaillées d'une verticale
  • get_trace — Consulter une trace d'audit par ID
  • get_benchmarks — Métriques de performance du Médiateur
